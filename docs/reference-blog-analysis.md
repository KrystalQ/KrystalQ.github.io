# 参考博客解析：one-null-pointer.github.io

参考站点：<https://one-null-pointer.github.io/>

## 1. 站点定位

参考博客是一个基于 Hexo 5.4.2 与 Fluid 1.9.2 主题搭建的个人技术博客。

它的核心定位不是作品集、官网或营销页，而是一个长期维护的私人学习档案。内容覆盖 AI、RAG、Dify、OpenClaw、Python、运维、安全、计算机基础、日记等主题。归档页显示已有 167 篇文章，说明站点价值来自持续记录与知识沉淀。

整体气质可以概括为：

- 私人领地感
- 技术学习笔记
- 长期归档
- 轻社交连接
- 阅读优先

## 3. 首页结构

首页由两个强烈区块组成。

### 3.1 首屏封面

首页首屏高度为 100vh，使用大图背景 `/img/default.png`，叠加 30% 黑色遮罩。中间只有一句打字机文案：

```text
你好，欢迎来到撩月的私人领地_(:зゝ∠)_
```

这一句是整个站点气质的关键。它不是功能介绍，而是空间宣言：访问者进入的不是文档库，而是一个人的私人领地。

### 3.2 文章卡片列表

首屏下方是文章列表。文章卡片使用左右结构：

- 左侧：文章封面图
- 右侧：标题、摘要、日期、分类
- 摘要限制 3 行
- 图片高度约 10rem
- 卡片之间留白充足

文章列表包裹在一个 `board` 容器中。这个容器从首屏下方略微上浮，具有圆角和阴影，形成“内容从封面浮出”的视觉层次。

## 4. 页面模式

### 4.1 文章详情页

文章页首屏高度为 70vh。封面中显示：

- 文章标题
- 发布时间
- 字数
- 阅读时长

正文区域采用三栏结构：

```text
左侧留白 2/12
正文区域 8/12
右侧目录 2/12
```

文章体验完整：

- Markdown 正文
- 代码高亮
- 代码复制
- 图片懒加载
- 图片点击放大
- 标题锚点
- 右侧目录
- 文章版权信息
- 上一篇/下一篇

适合长技术文章与代码笔记。

### 4.2 归档页

归档页是纯列表结构：

```text
167 posts in total
2026
03-15  AI--智能体Reflection架构
03-13  AI--智能体Plan-and-Solve架构
...
```

它几乎没有装饰，重点是高效回溯。归档页的价值是时间线，而不是视觉展示。

### 4.3 分类页

分类页使用可折叠面板：

```text
AI        15
RAG       5
DIFY      1
OpenClaw  2
Python    7
运维体系   30
...
```

点开分类后，显示该分类下的文章列表。这个设计非常适合文章较多的知识库，因为读者能快速判断作者主要积累在哪些领域。

### 4.4 关于页

关于页非常私人化：

- 圆形头像
- 站名
- 网易云音乐播放器
- 自嘲式简介
- QQ 联系方式
- 日期

它不像正式简历，更像一段“主人留言”。这给技术博客增加了人味。

### 4.5 友链页

友链页是三列卡片网格：

- 圆形头像
- 昵称
- 一句短介绍
- hover 时头像轻微放大

它传达的是“我属于一个小圈子”，而不是复杂社交。

## 5. 视觉系统

参考站的视觉来自 Fluid 主题，整体为灰蓝、白、浅灰、深灰的低饱和系统。

浅色模式：

```text
页面背景：#eee
内容板：#fff
主文字：#3c4858
次文字：#718096
导航背景：#2f4154
链接 hover：#30a9de
```

深色模式：

```text
页面背景：#181c27
内容板：#252d38
主文字：#c4c6c9
次文字：#a7a9ad
导航背景：#1f3144
```

视觉特点：

- 灰蓝调，不刺眼
- 内容容器阴影明显
- 圆角较小
- 阅读区域稳定
- 装饰性弱，内容优先
- 封面图承担主要氛围塑造

## 6. 字体与排版

站点使用系统 sans-serif 字体，整体朴素、稳定。全局有轻微字间距：

```css
html {
  font-size: 16px;
  letter-spacing: 0.02em;
}
```

文章正文使用 `github-markdown-css` 风格，适合代码、表格、列表和技术文档。

正文排版特征：

- 1rem 基础字号
- 行高约 1.6
- 标题加粗
- 代码块有行号和复制按钮
- 图片居中、圆角、阴影
- 引用和辅助文字使用次级灰色

## 7. 交互系统

参考站的交互都服务阅读：

- 顶部导航随滚动收缩并变背景
- 打字机标题
- 页面加载进度条
- 返回顶部按钮
- 本地搜索弹窗
- 深色模式切换
- 图片懒加载
- 图片灯箱放大
- 代码复制
- 文章目录跟随
- 标题 hover 显示锚点
- 移动端汉堡菜单

这些交互不是为了炫技，而是为了让长期阅读、查找、复制代码更顺手。

## 8. 内容组织方式

文章标题常用模式：

```text
主题--具体内容
```

例如：

```text
AI--智能体Reflection架构
RAG--RAG离线流程开发
Dify--Dify部署
Python--函数基础
```

优点：

- 主题识别快
- 分类感强
- 适合系列文章
- 适合归档列表快速扫描

正文写法偏学习笔记：

- 从“前言”开始
- 逐节解释概念
- 配代码、图和流程
- 常带参考来源
- 语气个人化，不完全学术

## 9. 可借鉴点

建议保留：

- 大封面建立个人空间感
- 单字中文导航
- 首页文章卡片列表
- 归档页时间线
- 分类页可展开列表
- 文章页右侧目录
- 搜索、深色模式、返回顶部
- 关于页保留私人表达
- 友链页保留轻社交关系
- 标题使用“主题--文章名”的系列化方法

## 10. 不建议照搬点

建议避免：

- 直接套 Fluid 主题导致撞站
- 过度依赖 Bootstrap 和外部 CDN
- 关于页使用内联 CSS、center 标签等旧写法
- 首页文章卡片略普通，可以重新打磨
- 字体识别度较弱
- 归档和分类页层次可以更清楚
- 默认背景图太通用，需要更贴合个人气质

## 11. 对新博客的设计启发

参考博客真正值得继承的不是某个组件，而是它的内核：

```text
私人领地感 + 长期学习归档 + 技术文章阅读体验
```

新博客应该延续这个内核，同时建立更明确的个人视觉：

- 更原创的首屏表达
- 更细腻的中文排版
- 更稳定的技术文章阅读体验
- 更适合未来长期扩展的信息结构
- 更少外部依赖
