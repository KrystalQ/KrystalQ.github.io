# 我的个人博客设计方案

## 1. 设计目标

这个博客不是展示型官网，也不是纯文档站，而是一个长期生长的个人知识空间。

设计目标：

- 让读者一进来就感到这是“我的私人知识场”
- 让技术文章足够好读，尤其是长文、代码、图片和系列内容
- 让归档、分类、搜索成为真正可用的知识入口
- 让视觉有个人气质，但不喧宾夺主
- 后续可以平滑迁移到 Astro、Hexo 或 GitHub Pages

核心关键词：

```text
私人领地
学习档案
技术长文
冷静克制
一点东方感
一点工程感
```

## 2. 设计概念

主题名：

```text
纸间旅栈
```

设计概念：

```text
像一家开在纸页之间的旅栈。
每篇文章是一间临时歇脚的房，每条笔记是一张夹在书里的路签。
这里不追求喧闹的入口，而追求一种可以停下来、翻一页、继续上路的秩序。
```

这个方向能继承参考站的“私人领地感”，但不再偏“个人书房”，而是偏“旅途中的知识驿站”：更轻、更有行走感，也更适合长期存放技术笔记、产品观察和生活札记。

## 3. 站点结构

一级导航不单独设置“首页”。左侧放 `纸间旅栈` logo，点击 logo 回到首页。

建议一级导航：

```text
某 / 卷 / 目 / 札 / 链
```

含义：

| 导航 | 页面 | 说明 |
|---|---|---|
| Logo：纸间旅栈 | Home | 回到首页，作为站点标识 |
| 某 | About | 关于我：带一点匿名旅人感，也不与参考站重复 |
| 卷 | Archives | 时间线归档：所有文章收成卷册 |
| 目 | Categories | 知识分类：像目录，也像门目 |
| 札 | Notes | 短笔记、碎片想法、踩坑记录 |
| 链 | Links | 友链与常看资源 |

这套命名与 `纸间旅栈` 的主题更一致：

```text
某 = 自我介绍
卷 = 文章归档
目 = 分类索引
札 = 路上随手记
链 = 朋友与资源
```

关于页的字定为 `某`。它比 `吾` 更有“某个旅人、某个写字的人”的匿名感，也更贴合 `纸间旅栈` 的停靠气质。备选可以是 `己`、`身`、`我`，但 `己` 偏内省，`身` 偏履历，`我` 太直白。

其中 `札` 建议保留，因为它能区分“正式文章”和“短记录”，更适合长期维护。

## 4. 首页设计

### 4.1 首屏

首页首屏不做传统 landing page，不放功能介绍。

结构：

```text
顶部固定导航：左侧 logo 回首页，右侧为某 / 卷 / 目 / 札 / 链
大幅背景图或生成视觉
站点名
一句打字机式私人宣言
最近状态
向下滚动提示
```

首屏文案建议：

```text
在混沌里写一点确定的东西。
```

备选：

```text
把学过的东西，折成一盏能回头照路的灯。
```

```text
这里存放代码、想法、复盘和一些还没想明白的问题。
```

首页标题建议直接使用主题名：

```text
纸间旅栈
```

### 4.2 首页内容区

首屏之后进入内容板块，保留“从封面浮出”的感觉，但减少传统卡片感。

内容结构：

```text
最新文章
  文章卡片列表
正在学习
  3 个当前关注方向
精选系列
  AI Agent / RAG / 产品分析 / 工程笔记
```

文章卡片建议：

- 左侧封面，不必每篇都大图，可以是主题色块或小图
- 右侧标题、摘要、日期、分类、阅读时长
- 技术系列文章使用统一主题标识
- 摘要最多 3 行

卡片布局：

```text
桌面：封面 30% + 内容 70%
移动：封面在上，内容在下
```

## 5. 文章页设计

文章页是整个博客最重要的页面。

结构：

```text
文章标题区
  标题
  分类
  日期
  阅读时间
  字数
正文区
  正文
  右侧目录
  代码复制
  图片放大
文章底部
  文章许可/作者信息
  上一篇/下一篇
  相关系列文章
```

建议增强：

- 右侧目录在桌面端 sticky
- 移动端目录收起为浮动按钮
- 代码块保留复制按钮
- 标题 hover 显示锚点
- 图片点击放大
- 文章底部显示“这篇属于哪个系列”

正文排版建议：

- 正文宽度：720px 到 780px
- 中文正文行高：1.85
- 英文和代码使用等宽字体
- 标题层级明显，但不要过大
- 表格横向可滚动
- 引用块使用左边线，不用重背景

## 6. 归档页设计

归档页保留参考站的效率，但增强时间线感。

结构：

```text
文章总数
年份导航
年份分组
  日期 + 标题 + 分类
```

示例：

```text
2026
03-15  AI--智能体 Reflection 架构      AI
03-13  AI--智能体 Plan-and-Solve 架构  AI
02-23  Dify--部署笔记                  DIFY
```

设计原则：

- 不做复杂卡片
- 重点是扫描效率
- 分类使用低调标签
- 年份要有明显分隔

## 7. 分类页设计

分类页做成“知识地图”。

结构：

```text
全部分类统计
分类列表
  AI Agent       15
  RAG            5
  产品分析        8
  工程笔记        12
  读书笔记        6
```

点击分类后展开文章列表。

设计增强：

- 分类按文章数量排序
- 每个分类显示一句说明
- 展开后最多先显示 8 篇，更多进入分类详情页

示例：

```text
AI Agent
关于智能体架构、工具调用、记忆、反思和评估。
15 篇
```

## 8. 关于页设计

关于页不要做简历页，要做“主人页”。

内容建议：

```text
头像或个人符号
一句自我介绍
当前关注
长期主题
联系方式
正在使用的工具
```

示例文案：

```text
我在这里记录技术学习、产品观察和一些没那么完整的想法。
没更新的时候，大概率是在学，还没整理好。
```

关于页可以保留轻松语气，但排版要比参考站更干净。

## 9. 链页设计

链页承载友链与常看资源，保留三列网格。

每个卡片：

- 头像
- 昵称
- 一句介绍
- 链接

可以分组：

```text
朋友
常看博客
工具与文档
```

这样链页不只是“朋友列表”，也能成为你的资源地图。

## 10. 视觉系统

### 10.1 色彩

建议主色不直接照搬参考站的灰蓝，而是做成“墨青 + 纸白 + 铜色点缀”。

浅色模式：

```text
背景：#eef1ef
内容：#fbfaf7
主文字：#17201c
次文字：#66736d
主色：#26443f
点缀：#b9823f
边线：#d7ddd7
代码背景：#101816
```

深色模式：

```text
背景：#0f1513
内容：#17201c
主文字：#dce6df
次文字：#9aa8a0
主色：#86b8a9
点缀：#d8a767
边线：#2a3934
代码背景：#0a0f0d
```

气质：

- 比参考站更温润
- 仍然适合技术内容
- 不落入常见紫蓝渐变
- 有一点纸张和书房感

### 10.2 字体

如果使用在线字体：

- 中文正文：Noto Serif SC 或 LXGW WenKai
- 英文标题：Cormorant Garamond 或 IBM Plex Serif
- 代码：Cascadia Code 或 JetBrains Mono

如果避免外部依赖：

```css
font-family: "Noto Serif SC", "Source Han Serif SC", "Songti SC", serif;
```

代码：

```css
font-family: "Cascadia Code", "JetBrains Mono", Consolas, monospace;
```

### 10.3 圆角与阴影

遵循参考站的小圆角，但更克制：

```text
容器圆角：8px
按钮圆角：6px
图片圆角：6px
阴影：轻，不要厚重
```

内容区不做太多卡片嵌套。文章卡片可以有分隔线，而不是每篇都一个重阴影盒子。

## 11. 动效

保留：

- 打字机首屏文案
- 导航滚动变色
- 返回顶部
- 搜索弹窗
- 深色模式渐变过渡
- 卡片 hover 轻微位移或封面缩放

避免：

- 大面积粒子背景
- 过多滚动动画
- 复杂加载动画
- 影响阅读的漂浮装饰

动效原则：

```text
首屏有记忆点，正文保持安静。
```

## 12. 内容模型

文章字段建议：

```yaml
title: AI--智能体 Reflection 架构
date: 2026-03-15
category: AI
tags:
  - Agent
  - Reflection
  - LLM
cover: /images/covers/ai-reflection.jpg
description: 让智能体从一次输出走向持续改进。
series: AI Agent 架构学习
```

分类建议：

```text
AI
RAG
产品分析
工程笔记
读书笔记
生活札记
```

文章标题建议延续参考站的系列化命名：

```text
AI--智能体 Reflection 架构
RAG--离线索引流程
产品分析--某某工具体验
工程笔记--一次部署问题复盘
```

## 13. 技术方案建议

短期可以使用纯静态 HTML 原型。

正式长期维护建议使用 Astro：

- Markdown/MDX 写文章
- 静态生成
- GitHub Pages 部署
- 支持 RSS
- 支持 sitemap
- 支持代码高亮
- 后续可加搜索索引

目录建议：

```text
src/
  content/
    posts/
  layouts/
    BaseLayout.astro
    PostLayout.astro
  pages/
    index.astro
    about.astro
    archives.astro
    categories.astro
    links.astro
  components/
    Navbar.astro
    Hero.astro
    PostCard.astro
    SearchModal.astro
```

## 14. MVP 范围

第一版建议做：

- 首页
- 文章详情页
- 归档页
- 分类页
- 关于页
- 友链页
- 深色模式
- 本地搜索
- 代码高亮
- 右侧目录
- GitHub Pages 部署

暂不做：

- 评论系统
- 后台管理
- 复杂标签云
- 访问统计
- 音乐播放器自动播放

## 15. 最终设计方向一句话

```text
一家开在纸页之间的个人旅栈：入口有停靠感，正文够安静，归档够清楚，长期写下去不会腻。
```
